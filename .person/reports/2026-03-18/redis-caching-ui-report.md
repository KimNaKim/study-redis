# 리포트: Redis 캐싱 서비스 및 시각화 UI 구현

**Date**: 2026-03-18  
**Reporter**: KimNaKim  
**Status**: [x] 태스크 5 완료, [x] 태스크 6 완료

## 1. 작업 요약
- Redis 캐싱 로직(Look-aside 패턴)을 구현하고, 이를 프론트엔드에서 실시간 애니메이션으로 확인할 수 있는 대시보드를 구축했습니다.
- 캐시 미스 시의 성능 차이를 체감할 수 있도록 인위적인 지연 시간을 적용했습니다.

## 2. 변경 사항
- **ProductService.java**: Redis 확인 -> DB 조회(1.5s 지연) -> 캐싱(TTL 60s) 로직 구현.
- **ProductController.java**: 상품 목록 조회 및 상세 정보 API 제공.
- **product-list.mustache**: 
    - Vanilla JS를 이용한 비동기 요청 및 상태 로깅 터미널 구현.
    - Bootstrap 기반의 로딩 애니메이션 및 데이터 출처 배지 표시.
- **CacheResponse.java**: 데이터 출처와 실행 시간을 포함한 공통 응답 객체 추가.

## 3. 검증 결과
- `/products` 페이지에서 첫 클릭 시 1.5초 지연(DATABASE), 재클릭 시 즉시 응답(CACHE)을 육안으로 확인했습니다.
- **비유로 설명**: 마치 매번 창고(DB)에 가서 물건을 꺼내오느라 시간이 걸리던 점원에게 자주 찾는 물건을 담아두는 바구니(Redis)를 준 것과 같습니다. 첫 주문은 창고에 다녀오느라 느리지만, 다음 주문부터는 바구니에서 바로 꺼내주어 속도가 매우 빨라진 것을 눈으로 볼 수 있게 한 것입니다.
