# Phase 4 Report: Sorted Sets (실시간 리더보드)

**Reporter**: G (Git User)
**Date**: 2026-03-19

## 1. 작업 요약
Redis의 **Sorted Sets(ZSet)** 자료구조를 활용하여 Score(조회수) 기반의 실시간 상품 인기 랭킹 시스템을 구축했습니다.

## 2. 주요 변경 사항
### Sorted Sets: 실시간 인기 랭킹
- **명령어**: `ZINCRBY`, `ZREVRANGEWITHSCORES`
- **로직**:
    - 상품 상세 조회 시 `ZINCRBY`를 호출하여 해당 상품의 Score(조회수)를 1점씩 증가시킵니다.
    - 메인 페이지 및 API 호출 시 `ZREVRANGE`를 사용하여 가장 점수가 높은 상위 5개 상품 정보를 실시간으로 추출합니다.
- **특징**: 데이터가 추가되거나 점수가 바뀔 때마다 Redis가 내부적으로 즉시 정렬을 수행하므로, O(log N)의 빠른 속도로 랭킹 정보를 유지할 수 있습니다.

### UI/UX 및 데이터 보강
- **실시간 리더보드**: 우측 사이드바에 'Top 5' 랭킹 섹션을 추가하고, 5초 주기로 자동 갱신(Polling)되는 로직을 적용했습니다.
- **데이터 확장**: `data.sql`에 3개의 상품을 추가하여 총 6개의 상품으로 더 실감 나는 랭킹 변동 테스트가 가능하도록 구성했습니다.
- **안정화**: `ProductService`에 누락되었던 랭킹 조회 로직을 보완하고, `ProductController`의 코드 구조를 표준에 맞춰 대대적으로 리팩토링했습니다.

## 3. 검증 결과
- **랭킹 변동**: 특정 상품의 'Fetch Detail'을 반복 클릭할 경우, 해당 상품의 순위가 실시간으로 상승하는 것을 확인했습니다.
- **자동 갱신**: 페이지 새로고침 없이도 5초마다 최신 랭킹 정보가 반영되는 것을 확인했습니다.
- **코드 무결성**: 컨트롤러의 중복 매핑 및 서비스의 메서드 누락 문제를 모두 해결하여 정상 기동을 확인했습니다.
