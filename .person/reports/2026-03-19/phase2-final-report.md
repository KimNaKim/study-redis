# Final Report: Phase 2 - Hashes (객체 필드 기반 관리)

**Date**: 2026-03-19  
**Reporter**: G  
**Status**: COMPLETED ✅

## 📝 작업 성과 요약
Phase 2에서는 Redis의 `Hashes` 데이터 구조를 도입하여, 기존 Strings 방식의 한계를 극복하고 효율적인 필드 단위 업데이트 및 조회를 구현했습니다. 특히 실시간 인터랙션과 벤치마크 기능을 통합하여 기술적 이점을 시각적으로 증명했습니다.

## 🚀 주요 구현 기능
1. **Hash 기반 캐싱**: `HSET`, `HGETALL`을 사용하여 객체를 필드 단위로 저장/조회.
2. **부분 업데이트 (Partial Update)**: 객체 전체를 다시 직렬화하지 않고 `price` 필드만 즉시 수정.
3. **원자적 재고 관리**: `HINCRBY`를 활용하여 동시성 이슈 없이 안전하게 재고를 차감.
4. **성능 벤치마크**: 1,000회 업데이트 시 Hashes 방식이 수배 빠름을 직접 증명.
5. **UI 고도화**: 터미널 스타일 로그, 애니메이션 효과, 캐시 수동 삭제 기능을 포함한 대시보드 구축.

## 📊 기술적 통찰 (Technical Insights)
- **효율성**: 전체 직렬화 비용이 제거됨에 따라 대규모 업데이트 환경에서 Hashes가 압도적으로 유리함을 확인했습니다.
- **정합성**: 캐시 업데이트 시 관련 Strings 캐시를 함께 삭제하여 데이터 불일치 문제를 해결했습니다.
- **원자성**: Redis 서버 레벨의 증감 연산을 통해 애플리케이션 레벨의 복잡한 락(Lock) 없이도 정확한 데이터 처리가 가능해졌습니다.
